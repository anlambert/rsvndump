Welcome to the rsvndump web page!

rsvndump is a command line tool to dump a subversion repository that resides on a remote server. All data is dumped in the format that can be read an written by svnadmin dump, so the data which is produced can easily be importerd into a new subversion repository.

Actually, rsvndump can basically be simulated with svnsync and a svnadmin dump afterwards. However, svnsync can not be used to "copy" single subdirectories of a repository. Since svndumpfilter can do this job on a dump, this is not an insuperable issue. But if you do not have access to the repository root or do not want to send the whole repository to your backup PC, rsvndump is the right tool for you.

rsvndump is written in C and built on top of the subversion API, so it can offer all functionality needed to access a subversion repository, including SSL authentication. At the moment, there is still need for the svn program to produce a log of the revisions that should be dumped (this will change soon). And it's GPLed.
